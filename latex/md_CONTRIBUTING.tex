\doxysection*{Submitting a pull request}

\href{https://github.com/Open-Acidification/TankController/pulls}{\texttt{ Pull Requests}} are used for adding new code and documents to the repository, or editing the existing code or documentation.


\begin{DoxyEnumerate}
\item \href{https://github.com/Open-Acidification/TankController/fork}{\texttt{ Fork}} and clone the repository
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Create a new branch\+: {\ttfamily git checkout -\/b my-\/branch-\/name}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Make your change
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Push to your fork and \href{https://github.com/Open-Acidification/TankController/compare}{\texttt{ submit a pull request}}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Pat your self on the back and wait for your pull request to be reviewed and merged.
\end{DoxyEnumerate}

Here are a few things you can do that will increase the likelihood of your pull request being accepted\+:


\begin{DoxyItemize}
\item Keep your change as focused as possible. If there are multiple changes you would like to make that are not dependent upon each other, consider submitting them as separate pull requests.
\item Write \href{http://tbaggery.com/2008/04/19/a-note-about-git-commit-messages.html}{\texttt{ good commit messages}}.
\end{DoxyItemize}

Draft pull requests are also welcome to get feedback early on, or if there is something blocking you.


\begin{DoxyItemize}
\item Create a branch with a name that identifies the user and nature of the changes (similar to {\ttfamily user/branch-\/purpose})
\item Open a pull request
\end{DoxyItemize}
